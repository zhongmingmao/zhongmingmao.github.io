# CLI Agent

> AI 原生开发的最佳载体 - **执行深度** + **集成自由度**

## 能力边界

> 不同 AI 交互形态的能力边界

![image-20260104234916507](https://ai-native-1253868755.cos.ap-guangzhou.myqcloud.com/image-20260104234916507.png)

<!-- more -->

1. 像 **Cursor** 和 **GitHub Copilot** 这样的 L2 工具也在快速进化，推出 Agent 模式
   - 在 **IDE 内部**提供了更强的**上下文感知**和**自动化能力**
2. 与 **Claude Code** 这类真正的**命令行原生智能体**相比
   - 最核心的区别：**执行环境的受限**
3. **Cursor** 和 **GitHub Copilot** 的**行动**被**绑定**在了运行了 IDE 的**本地开发环境**中
   - 而一个真正的 **CLI Agent**，则是可以被部署到**任何目标环境**中
   - 包括但不限于：**服务器**、**CI Runner**、**Docker 容器**

## 原生工作流智能体

1. CLI AI Agent - **L3/L4 成熟度**，<u>与开发者共享终端</u>
   - 一切皆**文件**、一切即可**由命令驱动**
2. 拥有无与伦比的**集成自由度**和**执行深度**

| 优势                 | 描述                                                         |
| -------------------- | ------------------------------------------------------------ |
| **完全的环境感知**   | 可以使用 ls tree git 等命令，来<u>全面地理解一个项目</u>     |
| **无限制的行动能力** | 理论上，CLI Agent 在被授权后，可以<u>执行任何 Shell 命令</u><br />可以参与到开发、测试、构建、部署的<u>任何一个环节</u> |
| **天然的可编程性**   | 本身是一个 CLI 程序，可以被轻松地集成到任何<u>脚本</u>、<u>Makefile</u>、<u>Git Hooks</u> 或 <u>CI/CD 流水线</u>中，成为<u>自动化流程</u>的<u>原子能力</u> |

> **CLI AI  Agent** 是实践 **SDD**、实现真正 **AI 原生工作流**的**唯一正确路径**

## 设计哲学

| CLI AI Agent | 设计哲学                                                     |
| ------------ | ------------------------------------------------------------ |
| Claude Code  | <u>为**自治**而生的**工作流引擎**</u>，设计核心为**智能体自治**<br />Claude Code 并非被设计成一个简单的问答工具，而是被构建用来**深刻理解项目上下文**、并具备**前瞻性地规划和执行复杂任务**<br />一开始就内置 **Hooks**、完整的**仓库导航**能力、**Git** 集成等丰富功能<br />目标：成为一个可以**独立规划**、**执行**、甚至**重试**多步**工作流**的**虚拟团队成员** |
| Gemini CLI   | **灵活**的**对话式上下文引擎**，设计核心为**流式**的、**富上下文的交互**<br />得益于 **Gemini Pro 模型**宣称的**巨大上下文窗口**和与 **Google 搜索**的**天然集成**<br />极其擅长处理**大规模上下文的重构**和**需要外部知识检索**的任务<br />设计哲学：可以随时调动**海量信息**并进行**深度对话**的超级大脑，但<u>并非一个完全自主的行动者</u> |
| Codex CLI    | **安全第一**的**精准编辑助手**，设计哲学是**安全**和**清晰**<br />专注于 **patch-based** 和 **diff-based** 的**文本编辑**<br />在执行任何修改前，都会以最清晰的方式**展示变更**并**等待批准** - 内置强大的**沙箱安全机制**<br />但同样牺牲了**自主规划**和**扩展**的能力 |

# Claude Code

## 完整方法论

1. 掌握一整套 AI 原生开发的**工作流**和**工程思想**
2. Claude Code 是**唯一一个**将 <u>AI 原生开发工程思想</u>实现得**最完整**、**最体系化**的工具
   - 上下文管理 CLAUDE.md + 安全控制 + 能力扩展 Hooks/MCP + 自主规划 + ...

## 事实标准

1. Claude Code 是 **AI Coding 领域的开创者**和**游戏规则的定义者**
2. Claude Code 是**业界第一个**真正意义上的 **CLI Coding Agent**
3. 许多**交互模式**，其最初的**范式**定义和推广都源于 Claude Code - 深入理解**交互范式**的**设计精髓**
   - @ 注入文件上下文
   - ! 执行 Shell 命令
   - / 发起斜杠指令

## 能力天花板

1. **Sub-agent**、**Hooks**、**Checkpointing** ... 这些目前仍然是 Claude Code **独创**或者**独有**的高级特性
2. **业界最高水准的功能** - 用于构建**复杂**的、**健壮**的自动化系统

## 生态繁荣

1. **先发优势**、**强大能力**和**广泛的用户基础** - 使得 Claude Code 成为**使用最多**、**用途最广**的 **CLI Coding Agent**
2. **正向的生态循环** - 智谱 AI 支持 Claude Code

# 分层配置

> 高层级配置会**覆盖**低层级配置中的**同名**配置

![image-20260106162803253](https://ai-native-1253868755.cos.ap-guangzhou.myqcloud.com/image-20260106162803253.png)

| 配置                                            | 描述                                                         |
| ----------------------------------------------- | ------------------------------------------------------------ |
| 企业级策略<br />managed-settings.json           | 由 <u>IT/DevOps</u> 团队统一分发，用于**强制执行**公司级安全策略<br />拥有<u>最高优先级</u>，**不可被覆盖** |
| 命令行参数                                      | 为**单次会话**提供的**临时覆盖**，适合<u>快速测试</u>        |
| 项目级个人配置<br />.claude/settings.local.json | 在此项目的特定偏好，**默认被 Git 忽略**，不会与<u>团队</u>共享 |
| 项目级共享设置<br />**.claude/settings.json**   | 需要**团队所有成员共享**的项目级规范（如权限规则），应该**提交**到**代码块** |
| 用户级全局设置<br />~/.claude/settings.json     | 存放<u>个人</u>的、在所有项目中都生效的全局配置              |
