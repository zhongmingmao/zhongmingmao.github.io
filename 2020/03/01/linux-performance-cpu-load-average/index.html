Linux性能 -- CPU -- 平均负载

Created2020-03-01|LinuxPerformance
|Word count:1.9k|Reading time:9min

uptime

1分钟、5分钟、15分钟

平均负载

  1. 单位时间内，系统处于可运行状态和不可中断状态的平均进程数，也就是平均活跃进程数，它和CPU使用率并没有直接关系
  2. 所谓可运行状态的进程，是指正在使用CPU或者正在等待CPU的进程
    • 也就是我们常用ps命令看到的，处于R状态（Running或Runnable）的进程
  3. 不可中断状态的进程则是正处于内核态关键流程中的进程，并且这些流程是不可打断的
    • 比如最常见的是等待硬件设备的I/O响应
      • 当一个进程向磁盘读写数据时，为了保证数据的一致性
      • 在得到磁盘回复前，它是不能被其他进程或者中断打断的，这个时候的进程就处于不可中断状态
    • 也就是我们在ps命令中看到的D状态（Uninterruptible Sleep，也称为Disk Sleep）的进程
    • 不可中断状态实际上是系统对进程和硬件设备的一种保护机制

理想情况

平均负载 = CPU个数

经验值

关注时机：平均负载 >= CPU * 70%

平均负载 / CPU 使用率

  1. 平均负载 – 平均活跃进程数
    • 单位时间内，处于可运行状态和不可中断状态的进程数
  2. CPU 使用率 – 单位时间内CPU繁忙情况的统计
  3. 两者不一定完全对应
    • CPU密集型进程，使用大量CPU会导致平均负载升高，此时这两者是一致的
    • I/O密集型进程，等待I/O也会导致平均负载升高，但CPU使用率不一定很高
    • 大量等待CPU的进程调度也会导致平均负载升高，此时的CPU使用率也会比较高

工具

工具 用途
stress Linux系统压力测试工具
sysstat 常用的 Linux 性能工具，监控和分析系统的性能
mpstat：多核CPU性能分析工具，用来实时查看每个CPU的性能指标，以及所有CPU的平均指标
pidstat：进程性能分析工具，用来实时查看进程的CPU、内存、I/O以及上下文切换等性能指标
