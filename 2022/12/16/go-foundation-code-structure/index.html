Go - Code Structure

Created2022-12-16|Go
|Word Count:2k|Reading Time:10mins

源文件

  1. Go 源文件使用全小写字母形式的短小单词命名，并以 .go 扩展名结尾
    • 如果使用多个单词，将多个单词直接连接起来，而不使用其它分隔符
    • 即 helloworld.go，而非 hello_world.go
  2. 尽量不要使用两个以上单词组合作为文件名，否则很难辨认

Hello, World

程序结构

  1. package 是 Go 语言的基本组成单位，通常使用单个小写单词命名
  2. 整个 Go 程序仅允许存在一个 main 包
  1. 当运行一个可执行的 Go 程序时，入口函数都为 main 函数
  2. import "fmt" - 包路径
    • 导入 fmt 包的包路径
    • 导入 Go 标准库下的 fmt 目录下的包
  3. fmt.Println - 将字符串输出到标准输出 - 包名
    • Println 函数位于 Go 标准库的 fmt 包中
    • 首字母大写的标识符为 Exported，对包外代码可见，否则为仅包内可见
    • main 函数中的 fmt 是一个限定标识符（Qualified identifier）
  4. gofmt - 标准 Go 代码风格使用 TAB 而不是空格来实现缩进

import - 包路径（唯一），函数内使用 - 包名（不唯一）

无法导入 main 包，会编译报错 - package main is not in std

Go 源文件本身采用 Unicode 字符集，并采用 UTF-8 字符编码
与编译后的程序所运行环境所使用的字符集和字符编码是一致的

结尾标识符 - ;

  1. Go 语言规范，使用 ; 作为结尾标识符
  2. 大多数 ; 是可选的，但在源码编译时，Go 编译器会自动插入这些被省略的 ;

Go Module

Go 是一种编译型语言：将源码编译生成可执行的二进制文件，并运行在没有安装 Go 的环境中

依赖第三方库：fasthttp 和 zap

直接 go build，编译失败，提示缺少 go.mod 文件

Go Module 是从 Go 1.11 正式引入的，用于解决 Go 项目复杂的依赖问题
从 Go 1.16 开始，成为默认的包依赖管理机制和源码构建机制

go.mod 存储了 Go Module 对第三方依赖的全部信息

Module 是 Package 的集合 - 这些 Package 和 Module 一起发布和分发

go.mod 所在的目录被称为所声明的 Module 的根目录

  1. module path - github.com/zhongmingmao/hello-go
  2. module 隐含 Namespace 的概念
    • module 下每个 package 的 import path，都是由 module path 和 package 所在的子目录构成的
    • 即 import path = module path + subdir

再次执行 go build，编译依然失败

自动添加依赖

  1. go.mod 记录了直接依赖和间接依赖的包的信息
  2. go.sum 记录了直接依赖和间接依赖的包的相关版本的哈希值，用于构建时校验本地包的真实性

再次执行 go build，编译成功
